폼(form) 태그

  • form 태그란? 사용자가 서버에 정보를 쉽게 전달할 수 있는 페이지를 구성함
  • BOM - document.form 객체
    정보를 입력받은 후 다른페이지로 정보를 전달함
    form태그의 정보를 배열로 저장하여 이름정보가 없어도 배열로 접근가능함

    document.forms[0].속성
    document.forms[0].기능()
    document.폼태그이름.속성

폼 태그 동작방법

  1. 폼이 있는 웹 페이지 방문한다.
  2. 폼 내용을 입력한다.
  3. 폼 안에 있는 모든 데이터를 웹 서버로 보낸다.
  4. 웹 서버는 받은 폼 데이터를 처리하기 위해 웹 프로그램으로 넘긴다.
  5. 웹 프로그램은 폼 데이터를 처리한다.
  6. 처리결과에 따른 새로운 html페이지를 웹 서버에 보낸다.
  7. 웹 서버는 받은 html페이지를 브라우저에 보낸다.
  8. 브라우저는 받은 html페이지를 보여준다.


폼 태그 속성 : name, action, mathod, target

  • action : 폼을 전송할 서버 쪽 스크립트 파일 지정
  • name : 폼을 식별하기 위한 이름 지정
  • accept-charset : 폼 전송에 사용할 문자 인코딩 지정
  • target : action에서 지정한 스크립트 파일을 현재 창이 아닌 다른 위치에 열도록 지정
  • method : 폼을 서버에 전송할 http 메소드를 정함.(GET 또는 POST)


페이지 전달방식(get/post)

get 방식 / get 메서드

  • 전송할 데이터(파라미터)를 URL의 정보에 붙여서 전송
  • 전송속도가 빠름 - body가 필요없기 때문
  • 데이터 길이제약 - 256byte 초과 불가
  • 적은량의 파라미터, 일반적인 데이터를 전달할 때 사용
  • 보안적으로 취약함 - 파라미터가 URL에 모두 보여지기 때문
    => 즉, 개인정보나 중요데이터는 전달하면 안됨
  • /itwill.html[?]ID=itwill[&]PW=123123

post 방식 / post 메서드

  • 전송할 데이터(파라미터)를 URL에 전달 X, 본문에 body에 붙여서 전송
  • 상대적으로 전송속도가 느림 - body 생성, 많은량의 데이터를 전달하기 때문
  • 데이터 길이제약 X - 파일업로드
  • 보안적으로 우수함 - 파라미터를 URL에 전달하지 않기 때문
    => 즉, 개인정보, 중요데이터 전달, 로그인할 때 사용
  • /itwill.html
